1. Browser-use：竞品创意信号采集机器人

Browser automationAI agentResearch workflow

browser-use 的机会不是“让 AI 控浏览器”，而是把 YouTube、Bilibili、Product Hunt、竞品官网和案例页变成可重复采集的创意信号入口。

解决的问题：日报容易偏 GitHub 项目本身，缺少真实用户如何使用、视频案例热不热、是否有可复用工作流。

对比：RSS 和 GitHub Search 只能看到项目更新；浏览器自动化更接近真实内容发现，但应作为候选信号采集器。

Codex 提示词

你现在是我的竞品创意信号采集机器人开发助手。请在当前仓库新增 research_agents/browser_signal_collector/spec.md，定义一个最小采集流程：输入关键词列表，分别搜索 YouTube、Bilibili、Product Hunt 和项目官网；输出字段必须包含 source、query、title、url、published_at、creator_or_author、use_case、evidence_text、possible_opportunity、dedupe_key。再新增 examples/browser_signal_collector/2026-07-10.sample.json，手写 5 条样例数据；新增 scripts/score_browser_signals.py，根据新鲜度、是否有具体案例、是否贴合 AI 视频/内容生产/Codex 工作流打分；最后生成 reports/browser_signal_collector_mvp.md，用中文说明今天如何 30 分钟验证。不要接真实浏览器，先把 schema、样例和评分跑通。

2. Maxun：非 GitHub 来源结构化采集器

Web dataAutomationSource pipeline

Maxun 的价值不是又一个爬虫工具，而是把网页变成结构化 API，适合做 AI Opportunity Radar 的非 GitHub 来源层。

你现在是我的非 GitHub 来源采集架构师。请在当前仓库新增 sources/web_extractors/schema.json，定义一个网页抽取器标准：source_name、source_url、page_type、refresh_frequency、selectors_or_rules、raw_fields、normalized_fields、dedupe_key、quality_signals、known_failures。再新增 examples/web_extractors/product_hunt_ai.sample.json、examples/web_extractors/youtube_search.sample.json、examples/web_extractors/bilibili_search.sample.json 三个样例；新增 scripts/normalize_web_candidates.py，把样例统一输出到 candidates/web/2026-07-10.json；最后生成 reports/web_source_extractor_plan.md，说明 Maxun 适合接哪些来源、不适合接哪些来源，以及如何避免低质量网页噪音进入 Top。

3. BrowserOS：本地私域研究浏览器工作台

Agentic browserPrivacyResearch workflow
